PX4 v1.16.1 · Pixhawk 6C Mini · src/modules/fw_pos_control（既存Takeoff拡張）

QGC Actuator Configuration Approach A Setup Required

尾翼ロックサーボ MAIN5 / MAIN6 の割り当て手順 — モーター MAIN3 は通常出力のまま

Approach A の要点： カタパルト処理は独立モジュールではなく既存の固定翼 Takeoff 制御（control_auto_takeoff()）に統合された。 モーター（MAIN3）は PX4 通常の推進出力（Motor 1）のままで、カタパルト拡張は 「尾翼解放までスロットルを idle 保持する」ゲートを掛けるだけ。 したがって Peripheral via Actuator Set を割り当てるのは MAIN5 / MAIN6 の 2 本だけ。
Signal flow — Approach A（CAT_EN = 1 のとき）
fw_pos_control (control_auto_takeoff)          FunctionActuatorSet            QGC output

  commandCatapultTailServos()   [DO_SET_ACTUATOR · param7=0]
  ├─ param1 ──────────────────────────→  Peripheral_via_Actuator_Set1  →  MAIN 5  [tail servo A]
  └─ param2 ──────────────────────────→  Peripheral_via_Actuator_Set2  →  MAIN 6  [tail servo B]

  TECS throttle (_att_sp.thrust_body[0])   [CAT_MOT_REQ_TAIL でゲート]
  └─ control allocation ──────────────→  Motor 1（通常の推進出力）     →  MAIN 3  [motor / ESC]

  尾翼解放まで thrust は FW_THR_IDLE に固定 → 解放後に通常の離陸スロットルへ。
  MAIN3 は QGC 上で既存の Motor 割り当てのまま（変更不要）。

Step C

Step D

既存 launch detector の設定（必須）

重要： Approach A は PX4 標準の launch detector を流用する。 RWTO_TKOFF が 1（滑走離陸）のままだとカタパルトのパスに入らず尾翼処理が動かない。 必ず RWTO_TKOFF = 0 にすること。
パラメータ名 設定値 意味・備考
RWTO_TKOFF 0 滑走離陸を無効化 → カタパルト / ハンドランチのパスを使う
FW_LAUN_DETCN_ON 1 加速度による launch 検知を有効化
FW_LAUN_AC_THLD 機体に合わせ調整 射出を検知する body-X 加速度しきい値（G ではなく m/s²）
FW_LAUN_AC_T 機体に合わせ調整 しきい値を連続超過する必要がある時間
手順

QGC 操作手順

  1. QGC を起動し、機体に USB 接続する 上部ステータスバーが緑になり MAVLink 接続が確立していること
  2. Vehicle Setup → Actuators タブを開く 左サイドバーのギアアイコン（⚙）→ "Actuators"
  3. MAIN Output リストで MAIN 5 / 6 の行を確認する 現在の Function 割り当てをメモしておく（ロールバック用）。MAIN3 は触らない
  4. MAIN5 / MAIN6 の Function ドロップダウンを変更する MAIN 5 → Peripheral via Actuator Set 1
    MAIN 6 → Peripheral via Actuator Set 2
    MAIN 3 → Motor 1（変更しない）
  5. MAIN5 / MAIN6 の Min / Max / Disarm / Failsafe PWM を設定する（Step B 参照） MAIN3 の ESC 校正値は変更しない
  6. Save を押してパラメータを書き込む 再起動不要。変更は即時反映される
  7. Parameters ページで CAT_* と FW_LAUN_* / RWTO_TKOFF を設定する CAT_EN は飛行テスト前まで 0 のまま。RWTO_TKOFF=0 を忘れずに
  8. Actuator Test で尾翼サーボ動作を確認する Safety スイッチを解除 → スライダーで MAIN 5 / 6 を動かしてロック↔リリースを確認
確認

セットアップ完了チェックリスト

  • MAIN 5 に Peripheral via Actuator Set 1 が割り当てられている
  • MAIN 6 に Peripheral via Actuator Set 2 が割り当てられている
  • MAIN 3 は既存の Motor 1 割り当てのまま（変更していない）
  • MAIN5 / MAIN6 に Min 1000 µs / Max 2000 µs / Disarm 1000 µs が設定されている
  • QGC Actuator Test で MAIN 5 / 6 サーボが動作する
  • CAT_TAIL5_LCK でサーボが実機のロック位置に合っている
  • CAT_TAIL5_REL でサーボがリリース位置まで動く
  • MAIN 6 の方向が MAIN 5 と逆の場合、CAT_TAIL6_LCK / REL を反転させている
  • RWTO_TKOFF = 0 / FW_LAUN_DETCN_ON = 1 になっている
  • CAT_MOT_REQ_TAIL = 1（尾翼解放までモーター idle 保持）
  • CAT_EN = 0 のまま（地上テスト完了後に 1 にする）
